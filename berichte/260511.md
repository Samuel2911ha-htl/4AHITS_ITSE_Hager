## Vorbereitung der Übungsumgebung

Betrachte folgendes Skript und finde heraus was es macht. Anschließend führe das Skript als User kali mit sudo aus, um die Laborumgebung einzurichten:

```bash
#!/bin/bash

# Gruppen anlegen
groupadd music
groupadd wrestle
groupadd emperors

# User anlegen und Gruppen zuweisen
useradd -m -G music elvis
useradd -m -G music,wrestle blondie
useradd -m -G wrestle hogan
useradd -m -G emperors nero

# Passwörter auf 'kali' setzen
echo "elvis:kali" | chpasswd
echo "blondie:kali" | chpasswd
echo "hogan:kali" | chpasswd
echo "nero:kali" | chpasswd

# Lab-Struktur vorbereiten
mkdir -p /tmp/lab/gigs /tmp/lab/drafts
echo "Draft" > /tmp/lab/drafts/lyrics.txt
echo "Plans" > /tmp/lab/wrestle_plans.txt
chown -R root:root /tmp/lab
chmod -R 755 /tmp/lab
```

Dieses Skript erstellt 3 Gruppen und 4 Benutzer, die Benutzer werden beim erstellen gleich Gruppen zu geordnet. Danach werden für alle Benutzer das Passwort auf `kali` gesetzt. Zuletzt wird ein Ordner erstellt mit Files und zwei Wörtern Text. Dann wird noch der Besitzer des Verzeichnis `/tmp/lab` auf root geändert. Mit dem Zusatz `chown -R` wird der Benutzer rekursiv, sprich auf alle Unterverzeichnise auf einmal geändert.

```bash
$ chmod 755 vorbereitung.sh

$ sudo ./vorbereitung.sh

$ getent passwd

kali:x:1000:1000::/home/kali:/usr/bin/zsh
elvis:x:1001:1004::/home/elvis:/bin/sh
blondie:x:1002:1005::/home/blondie:/bin/sh
hogan:x:1003:1006::/home/hogan:/bin/sh
nero:x:1004:1007::/home/nero:/bin/sh
```

## Übung (Analyse und Symbolic Mode)

1. **Rechte auslesen:** Ermittle die Permissions, den User Owner und den Group Owner der Datei   /tmp/lab/wrestle_plans.txt?
```bash
┌──(root㉿kali)-[/tmp/lab]
└─# ls -all

-rwxr-xr-x  1 root root   6 May 11 06:12 wrestle_plans.txt
```
2. **Private Files:** Erstelle eine Datei notes.txt. Ändere die Permissions so, dass der other-Klasse (o) das Read-Recht (r) entzogen wird. Nutze den symbolischen Modus von chmod.
```bash
┌──(root㉿kali)-[/tmp/lab]
└─# vim notes.txt

chmod o-r notes.txt
```
Mit `-` werden Berechtigungen entfernt.

3.  **Group Sharing:** Setze die Permissions für notes.txt so, dass die group (g) die Datei lesen (r) und schreiben (w) darf, während der user (u) alle Rechte behält und others (o) keinerlei Zugriff haben. Verwende für die Änderung einen einzigen chmod Aufruf.
```bash
chmod 760 notes.txt
```
rwx --> 4 | 2 | 1, user, group, other. User bekommt in Summe 7 für read, write, execute. Group 6 für r,w und Other bekommt keine Berechtigung also 0.

4.   **Skript-Vorbereitung:** Erstelle die Datei myscript.sh. Welchen Befehl nutzt du, um dem user (u) und der group (g) das Recht zu geben, dieses File auszuführen (x)?
```bash
vim myscript.sh

chmod 110 myscript.sh
```
Hiermit hat User und Group nur das Recht auszuführen, lesen und schreiben sind nicht erlaubt.

5.   **Kollektive Änderung:** Erstelle die Datei data.txt. Entziehe mit einem einzigen Befehl im symbolischen Modus sowohl der group (g) als auch others (o) die Write- (w) und Execute-Rechte (x) für die Datei data.txt.
```bash
chmod go-wx data.txt
```
Im symbolischen modus werden die buchstaben einfach hintereinander geschrieben.

## Übung (Ownership und Gruppenkollaboration)

1. Wechsle mit sudo su in eine root shell. Füge den User blondie so zu /etc/sudoers hinzu (mit visudo), dass dieser den Befehl mv als root ausführen darf.
```bash
sudo su root

sudo visudo
# In der Datei unter User privilege specification
blondie ALL=(ALL:ALL) /usr/bin/mv
```
Nun noch in nano speichern mit `STRG + X` und `Y`

2. Group Change: Wechsle zum User blondie (su - blondie). Erstelle die Datei song.txt in deinem Home-Verzeichnis. Verschiebe die Datei anschließend nach /tmp/lab/ (verwende dafür sudo). Ändere danach den Group Owner der Datei auf music.
```bash
su blondie

echo "Song" > ~/song.txt
sudo mv ~/song.txt /tmp/lab

chgrp music song.txt
```
Mit benutzen von `~` benutzt man sein eigenes Home Verzeichnis, darauf hat man Schreibberechtigungen.

3. Experimentiere mit dem Unterschied von su und su - bei wechsel auf einen anderen User. Hinweis: mit exit kehrt man zum vorhergehenden User zurück, d.h. die User Session bleibt geöffnet während man mit su zu einem anderen User wechselt.
```bash
sudo su kali
Sorry, user blondie is not allowed to execute '/usr/bin/su kali' as root on kali.
```
Beim Nutzen von su wenn man bereits in einem Benutzer ist erfordert auch die entsprechenden Rechte für den Befehl `su`.

4. User Transfer: Du bist als root angemeldet. Mache elvis zum User Owner der Datei /tmp/lab/song.txt

6. Einschränkungen: Du bist als User elvis angemeldet. Er ist Mitglied in music, aber nicht in emperors. Welchen Befehl müsstest du eingeben, um zu versuchen, den Group Owner eines Files auf emperors zu ändern? Warum schlägt dieser Befehl fehl?
```bash

```

Dieser Befehlt schlägt fehl, da ein User Files nur auf eine Gruppe vergeben darf in der er selbst Mitglied ist und diese Datei ihm gehört. Da die Files jedoch root gehören ist dies ein weiterer Grund warum es nicht funktionieren kann.
