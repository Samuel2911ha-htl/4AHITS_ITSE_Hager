## Übung (ssh auf Metasploitable)
Als Vorbereitung für die folgende Übung.

Konfiguriere passwortloses ssh Login auf Metasploitable mit Hilfe eines ssh Public/Private keys.

Um das Schlüsselpaar zu erstellen.
```
$ ssh-keygen
```

Jetzt muss der Public Key von Metasploitalbe in `authorized_keys` gespeichert werden um Kali mit dem erforderlichen Schlüssel dann Zugriff zu gewähren. Die benötigten Berechtigungen mit `chmod` nicht vergessen.
```
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

Um den Schlüssel nun auf Kali zu bekommen wird ssh verwendet da Metasploitable kein direktes Kopieren über STRG + V zulässt. `ssh -oHostKeyAlgorithms=+ssh-rsa -oPubkeyAcceptedKeyTypes=+ssh-rsa msfadmin@192.168.153.128`. Da Metasploitable die neuen default Formate wie ed255 nicht vesteht braucht man den Zusatz `-oHostKeyAlgorithms=+ssh-rsa -oPubkeyAcceptedKeyTypes=+ssh-rsa`, damit sich Kali trotzdem verbindet.
Nun kann der Schlüssel ausgelesen und kopiert werden.
```
cat ~/.ssh/id_rsa
```

Betrachte besonders die notwendigen Änderungen in `~/.ssh/authorized_keys`, diese Datei muss in der nachfolgenden Übung manipuliert werden.

Hinweis: der ssh-server auf Metasploitable unterstützt nur Schlüssel im RSA Format (nicht die neueren default Formate wie ed255…).
Der Kali ssh Client lehnt die Verbindung mit diesem alten Schlüssel standardmäßig ab, aber man kann es mit Optionen wieder erzwingen.
Bei ssh Verbindungsproblemen hilft häufig die Verwendung der Verbose Optionen des ssh clients (`-v`, `-vv` oder `-vvv`).
